# Teka

Teka — это SPA-приложение для хранения и управления пользовательскими данными в виде репозиториев и элементов внутри них.

Основная функция проекта: **хранение данных** локально в браузере (IndexedDB) с удобным интерфейсом для создания, редактирования, удаления и поиска.

## Что умеет приложение

- Создавать и удалять репозитории (коллекции).
- Редактировать репозитории: название, описание, цвет и фон.
- Хранить элементы внутри репозитория:
  - название,
  - заметка,
  - теги,
  - изображение.
- Редактировать и удалять элементы.
- Просматривать элементы в двух режимах: список и плитка.
- Фильтровать элементы по тегам.
- Автозаполнение демо-данными при первом запуске.

## Где хранятся данные

Данные сохраняются локально в браузере:

- **IndexedDB** (через `idb`) — основное хранилище репозиториев и элементов.
- **Pinia + persisted state** — сохранение состояния UI и выбранных данных между перезапусками.

Это означает, что приложение работает без серверной части, а данные остаются на устройстве пользователя.

## Технологии

- Vue 3 (Composition API)
- Vite
- Vue Router
- Pinia
- PrimeVue / PrimeFlex / PrimeIcons
- IndexedDB (`idb`)
- UUID
- ESLint, Prettier

## Структура проекта (основное)

```text
src/
  assets/            # стили и статические ресурсы
  components/        # переиспользуемые UI-компоненты
  composables/       # работа с БД и утилиты
  demo-data/         # демо данные
  layout/            # layout-компоненты
  pages/             # страницы приложения
  router/            # маршрутизация
  store/             # Pinia stores
```

## Установка и запуск

1. Клонировать репозиторий:
```bash
git clone https://github.com/AL5t/Teka.git
```

2. Перейти в директорию проекта:
```bash
cd Teka
```

3. Установить зависимости:
```bash
npm install
```

4. Запуск в режиме разработки:
```bash
npm run dev
```

5. Production-сборка:
```bash
npm run build
```

6. Локальный предпросмотр production-сборки:
```bash
npm run preview
```
